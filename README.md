Техническое задание: Веб‑приложение «Заметки» (Notes Web App)
1. Общее описание
Название проекта: Notes Web App.

Цель: разработать удобное веб‑приложение для создания, хранения и управления текстовыми заметками с возможностью синхронизации между устройствами.

Целевая аудитория:

студенты и школьники (конспекты, списки дел);

офисные сотрудники (рабочие задачи, идеи);

все, кто ведёт личный дневник или списки покупок.

2. Функциональные требования
2.1. Пользовательский аккаунт
регистрация по email/паролю;

вход/выход из системы;

восстановление пароля;

настройка профиля (имя, аватар).

2.2. Работа с заметками
создание новой заметки (заголовок + текст);

редактирование существующей заметки;

удаление заметки (с подтверждением);

поиск по заголовкам и тексту заметок;

фильтрация по дате создания/изменения;

сортировка (по дате, алфавиту);

тегирование заметок (добавление, удаление тегов);

архивирование заметок (перенос в архив без удаления);

восстановление из архива.

2.3. Визуализация и интерфейс
список заметок с превью (заголовок + первые 50 символов);

полный просмотр заметки по клику;

режим «полный экран» для редактирования;

тёмная/светлая тема интерфейса;

адаптивный дизайн (ПК, планшет, смартфон).

2.4. Синхронизация и экспорт
автосохранение изменений;

синхронизация между устройствами (через сервер);

экспорт заметки в PDF/TXT;

импорт заметок из TXT‑файлов.

3. Нефункциональные требования
Производительность: загрузка списка заметок — не более 1 с; сохранение изменений — не более 0,5 с.

Безопасность:

HTTPS;

хеширование паролей (bcrypt);

JWT‑аутентификация;

защита от XSS/CSRF.

Надёжность: резервное копирование данных раз в сутки.

Доступность: поддержка скринридеров, клавиатурная навигация.

Локализация: русский и английский языки.

4. Технологии и стек
Фронтенд: React 18 + TypeScript, SCSS, React Router.

Бэкенд: Node.js (Express/NestJS).

База данных: PostgreSQL (хранения заметок и пользователей).

Аутентификация: JWT + refresh‑токены.

Хранение файлов: локальная ФС для экспорта PDF/TXT.

Деплой: Docker + Nginx, хостинг — VPS/VDS.

CI/CD: GitHub Actions (тесты, деплой).

Тестирование: Jest (юнит), React Testing Library (компонентные).

5. API‑эндпоинты (пример)
Аутентификация
POST /api/auth/register — регистрация;

POST /api/auth/login — вход;

POST /api/auth/refresh — обновление токена;

POST /api/auth/logout — выход.

Заметки
GET /api/notes — список заметок (с пагинацией, фильтрами);

GET /api/notes/{id} — заметка по ID;

POST /api/notes — создание заметки;

PUT /api/notes/{id} — обновление заметки;

DELETE /api/notes/{id} — удаление заметки.

Теги
GET /api/tags — список тегов;

POST /api/notes/{id}/tags — добавление тега к заметке;

DELETE /api/notes/{id}/tags/{tag} — удаление тега.

7. Этапы разработки
Этап 1. Прототип (1 неделя)

настройка репозитория GitHub;

базовый фронтенд (список заметок, форма создания);

макет бэкенда (Express + PostgreSQL);

CRUD для заметок (без аутентификации).

Этап 2. Аутентификация (1 неделя)

реализация регистрации/входа;

JWT‑авторизация;

защита маршрутов.

Этап 3. Расширенная функциональность (2 недели)

тегирование и фильтрация;

поиск по заметкам;

архивирование;

экспорт в PDF/TXT.

Этап 4. UX/UI и адаптация (1 неделя)

тёмная тема;

адаптивный дизайн;

анимации переходов.

Этап 5. Тестирование и деплой (1 неделя)

юнит‑ и компонентные тесты;

нагрузочное тестирование;

деплой на сервер;

документация API (Swagger).

8. Критерии приёмки
Все API‑эндпоинты работают и возвращают корректные статусы.

Данные сохраняются в PostgreSQL.

Аутентификация защищена (JWT, хеширование).

Интерфейс адаптивен (ПК/планшет/смартфон).

Покрытие тестами — не менее 70 %.

Документация в README.md актуальна.

Приложение деплоится в один клик (Docker).
